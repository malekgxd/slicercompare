Status: Done (Approved)

## Story

As a print farm operator,
I want clear error messages and visual feedback for all operations,
so that I can quickly identify and resolve issues without confusion.

4. File upload errors clearly indicate the problem (size, format, permissions)
5. CLI errors translated from technical to user-friendly messages
6. Network/database errors handled gracefully with retry options
7. Progress indicators show percentage completion for long-running operations

### Error Categories
- **User Input Errors (400):** Invalid file, missing parameters, validation failures
- **System Errors (500):** Database errors, filesystem errors, unexpected crashes
- **External Errors (502/503):** CLI failures, network issues, timeout errors
- **Not Found (404):** Missing sessions, configurations, or files

### Toast Notification Patterns
- **Success:** Green checkmark, auto-dismiss after 3 seconds
- **Error:** Red X icon, dismissible, actionable message
- **Loading:** Spinner icon, persists until operation completes
- **Info:** Blue info icon, auto-dismiss after 5 seconds

### Loading State Patterns
- **Button Loading:** Disable button, show spinner, change text to "Processing..."
- **Page Loading:** Full-page overlay with spinner and message
- **Inline Loading:** Skeleton loaders for content that's being fetched
- **Progress:** Determinate progress bar for operations with known duration
